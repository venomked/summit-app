# CLAUDE.md — ひろしさんダイエットアプリ 保守指示書

このリポジトリは、ひろし（47歳・166cm・ERPエンジニア）の減量ダッシュボード（単一HTML・オフライン動作）です。
Claude Codeでの更新セッションは必ずこのファイルの規約に従うこと。

## アプリの目的
- 目標：76.3kg → 63.0kg（Phase1: 74kg台/2026-08末、Phase2: 68-70kg台/年内、Phase3: 63kg/2027前半）
- 毎朝の体重/体脂肪率/腹囲の記録、7日平均と週次判定、NIHホールモデルによる90日予測
- レディネス採点（自動調節）による当日メニュー生成、4週サイクル（第4週ディロード）
- 制約：四十肩（肩を使う種目は整形外科許可まで禁止）、23:30就寝が最優先

## 絶対に壊してはいけない不変条件
1. 外部通信ゼロ：fetch/XMLHttpRequest/sendBeacon/外部scriptタグを追加しない。CSPの connect-src 'none' を維持
2. 単一HTMLファイル完結（ビルド不要・Netlify Dropで配置できること）
3. localStorageキーの互換性維持：diet:latest / diet:hist / diet:checks / diet:cond / diet:start
4. ヘルスケア連携のURLパラメータ仕様維持：?w=&f=&c=（記録後にURLから消去）
5. 健康上の安全規則を緩めない：22:30以降・疲労高は運動オフ／肩の痛み強→整形外科案内／ふくらはぎ痛→ジョグ禁止

## 知識ベースの更新手順（「最新情報に更新して」と依頼された場合）
1. WebSearchで運動生理学・栄養学の最新ガイドライン/メタ分析を確認（一次情報優先）
2. 反映先は3か所のみ：
   - buildPlan() のルール（強度・警告のロジック）
   - renderForecast() のモデル係数（変更時は根拠を必ずコメントに）
   - 「拠」アコーディオンの根拠リスト（出典名＋年を必ず付記）
3. フッターの「知識ベース：最終更新」日付を更新
4. バージョンを上げ（eyebrowと更新ログ）、変更内容を更新ログに1行追記

## 品質ゲート（必須）
- 変更後は `npm install && npm test` を実行し、全件OKになるまで完了としない
- 新機能を追加したら対応するテストを app.test.js に追加する
- テストの期待値を安易に書き換えて通さない（挙動変更が意図的な場合のみ、理由をコミットメッセージに明記）

## デプロイ
- index.html を Netlify（同一サイトに上書き）へ。同一URLなら端末のlocalStorageデータはそのまま引き継がれる
- localStorageキーを変更した場合のみ、移行コードをloadState()に追加すること（v4.3の前例あり）

## 口調・表記
- UI文言は日本語・励まし基調・断定的な医療診断はしない（整形外科など受診案内に留める）
- 数値は単位付き、日付はYYYY-MM-DD
